Wine Dependencies

Wine has three build formats and depending on what build you want, you will have to pay attention to what you install and its format.

The three build types are: 32-bit, 64-bit, and WoW64. 32-bit and 64-bit are self-explanatory, but WoW64 needs explanation. WoW64 is a compatibility layer created by Microsoft when applications and computers were becoming 64-bit. This layer allows users to emulate 32-bit Windows applications on a 64-bit CPU. Knowing this, a Wine WoW64 build can be explained as a build type that allows the user to execute Windows 32-bit and 64-bit applications.

If you don't want a WoW64 build, follow the instructions normally and skip the lib32 instructions. When you reach the MinGW-w64 section, follow only the x86_64 instructions.

If you want a 32-bit only build, follow only the lib32 instructions. When you reach MinGW-w64, follow the i686 instructions. However, for the sake of continuing onto BLFS, it is recommended to do the normal installations as well, save for MinGW-w64.

And finally, if you want a WoW64 build, follow all the instructions on the dependency pages unless instructed otherwise or are presented with options.

Required

Audio (alsa-plugins-1.2.12 and/or PulseAudio-17.0), dbus-1.16.2, Display server (Xorg-Server-21.1.16 or Wayland-1.23.1 w/ Xwayland-24.1.6), Fontconfig-2.16.2, FreeType-2.13.3 - Pass 2, GnuTLS-3.8.9, libunwind-1.8.1 (if 64-bit/WoW64; not used by Wine 32-bit), OpenGL (libglvnd-1.7.0 or Mesa-25.1.2), and MinGW-w64-GCC-15.1.0

Recommended

  • GStreamer-Suite-1.26.2: allows consuming and working with multimedia such as music and videos. Most games and some software work without it, although some will need it for WMA decoding.

  • SDL2-2.32.8: for input and other neat stuff.

  • Vulkan-Loader-1.4.313: for Vulkan support.

Optional

libcapi, Cups, gphoto2, Samba, SANE, Kerberos, smbclient, OpenCL, libpcap, libusb, and v4l-utils
